# 神笔马良短篇小说 Agent

这是一个个人自用、本地运行的 AI 短篇小说创作工作台。当前版本已经把工作台、写作流程、后端接口、本地工程结构和 Kimi/DeepSeek/OpenAI/模拟内核跑通。

## 仓库说明

- 目标仓库：`franciscogeremias397-ux/writeagent`
- 产品形态：本地优先的个人写作工作台，不是 SaaS、多用户平台、移动 App、桌面 App 或浏览器扩展。
- 本地入口：Web 默认运行在 `http://localhost:3000`，API 默认运行在 `http://localhost:3001/api`。
- 隐私边界：`.env`、本地作品数据、日志、备份、截图、上传文件和运行产物不会提交到 Git。
- 详细需求：见 `docs/requirements.md`。
- Codex 任务模板：见 `prompts/codex-task-template.md`。

## 你现在能看到什么

- 首页创作驾驶舱：读取后端作品和趋势数据，CSV 导入的新题材会同步出现在首页推荐里；今日灵感可以切换，并可一键带入灵感写作页；本周创作进度会按草稿/连载作品的真实更新时间和字数计算，并给出继续写作入口
- 灵感写作：输入一个点子，按选题卡、情绪曲线、冲突阶梯、信息差、人物卡、场景卡、场景提示词、分场正文、测试读者报告、修订建议生成完整短篇方案
- 自动写作：输入一句创作方向即可按十步生成短篇草稿；平台、赛道、篇幅、情绪、主角、结局和文风可以在高级参数里继续调
- 灵感和自动写作参数可以保存为本地记录，刷新或重启后仍能继续套用
- 步步确认：生成结果可以按“确认选题 → 情绪曲线 → 冲突阶梯 → 信息差 → 人物卡 → 场景卡 → 场景提示词 → 分场正文 → 读者评审 → 修订建议”逐段检查，也可以选择另一张选题卡重新生成后续内容
- Agent 调度轨迹：每次生成都会记录主控 Agent 如何把任务交给风向分析、选题、结构、场景卡、提示词、正文、测试读者、编辑改稿和复盘沉淀 Agent；保存作品后在作品详情和导出的工程目录里也能回看
- 写作内核有分阶段合同：整篇正文必须由分场正文合并而来，场景卡、场景提示词、分场正文必须一一对应，真实 AI 返回结果也会经过这道校验
- 生成结果会展示每个场景的写作提示词、正文片段、小评分和修改重点；保存前可以只重写某一场，不用整篇推倒重来
- 生成结果会附带质量体检，检查开头钩子、中段拖沓、信息差回收、场景推进、套路化风险和结尾后劲，并保留发布前合规边界提醒
- 生成结果会附带原创边界报告，拆清“可学习点”和“避免复制点”，提示哪些人物关系、桥段、信息差和空话水文风险需要原创化替换
- 生成结果会附带作品记忆，记录人物状态、人物关系变化、已埋伏笔、待回收伏笔、每场连续性提醒，避免单场重写时把人物线和伏笔改断
- 风向标：读取后端趋势数据，查看热门赛道、活动/征文方向、风险提醒、机会卡和榜单作品拆解；CSV 导入的新题材会出现在这里，并可一键带入自动写作
- 作品专栏：书架式管理作品，支持手动建档、编辑资料、删除作品、按状态筛选、按作品名/赛道/标签/平台搜索，并可进入单部作品详情
- 作品详情：集中查看作品总览、数据表现、故事梗概、情绪曲线、冲突阶梯、信息差、人物关系图谱、人物卡、场景卡、场景提示词、分场正文、测试读者报告、修订建议和正文草稿；正文页会显示待处理标记、已应用改写版本，也可以只重写某一场并保存回作品；记忆策略页会同时显示相关写作记忆和个人策略；复盘页可以直接带着当前作品去生成或更新复盘，并把复盘建议带入自动写作生成下一篇
- 正文编辑器：切换已保存作品，打开真实正文，选中文本添加标记、@标记反馈、生成或重新生成局部改稿、应用修改、记录原文和新文版本
- 数据看板、复盘分析、写作记忆库、数据源管理、设置中心；数据看板会同时展示平台风向拆解、活动窗口、题材机会表现、作品收益/完读拆解和评论关键词，设置中心会展示 AI 状态、本地运行体检、本地存储目录、采集设置、采集边界和备份恢复
- 有 Kimi、DeepSeek 或 OpenAI Key 时优先走真实 AI；没有 Key 时自动使用本地模拟内核
- 生成结果可以点击「保存为作品」，数据库可用时会写入本地 PostgreSQL；保存时会把测试读者建议同步沉淀成一条写作记忆，下一篇写作可以继续参考
- 保存作品时会一并保存当次生成的完整写作方案；PostgreSQL 可用时写入数据库，数据库不可用时写入本地文件。后续在作品详情里重写单场、导出作品工程时都会优先使用这份方案
- 编辑器标记、应用改稿后的版本历史和改完的作品全文，数据库可用时会写入本地 PostgreSQL
- 应用改稿时会先询问是否更新写作记忆，可勾选人物状态、人物关系、伏笔、情绪弧线、用户偏好和后续注意事项
- 复盘分析可以录入阅读、收益、完读率、排名变化、推荐量变化和评论反馈，生成作品复盘报告、内容维度诊断，并把可复用经验沉淀到写作记忆库和个人策略库；生成复盘后可以一键“用复盘写下一篇”，自动把平台、题材和下一篇建议带入自动写作
- 灵感写作和自动写作会在生成前读取已启用的写作记忆和个人策略，并在结果里显示本次参考了哪些经验
- 作品专栏可以导出单部作品工程目录，文件会写入 `workspace/works`
- 数据源管理可以新增数据源、填写来源地址/文件说明、设置更新频率和启用状态，并能在列表里直接启用/停用数据源；也可以启动本地浏览器采集番茄作者后台可见页，用户自行登录和处理验证码后，系统只读取当前页面已经显示的作品表现、评论反馈、榜单和活动信息，不保存账号、密码或 Cookie；公开页面数据源可以在列表里直接点「采集」，CSV 会在导入前展示字段识别、字段映射和前几行预览，选择本地 CSV 文件导入时会把原件留存在 `storage/uploads/csv`，可识别的题材热度会进入趋势数据，CSV、平台文字和截图识别/校正文字里的作品阅读、收益、完读率、评论反馈和评论关键词会回写到作品库，并自动沉淀为写作记忆和个人策略，反哺下一篇的创作依据；单独上传截图时，有 Kimi/OpenAI Key 会自动识别文字，没有 Key 会先保存原图并登记等待校正任务，之后可在等待卡片里直接补文字；公开网页采集失败时，日志里保留来源地址并可一键重试
- 保存过的灵感和参数模板会写入 `storage/local-data/writing-assets.json`，并会进入本地备份

## 最简单的打开方式

在 Mac 上可以直接双击项目根目录里的这 3 个文件：

- `启动神笔马良.command`：自动准备本地目录、依赖、数据库可用时启动数据库，然后打开 `http://localhost:3000`
- `停止神笔马良.command`：停止本地网页服务和 Docker 里的数据库/队列服务，不会删除作品和备份
- `体检神笔马良.command`：检查本地配置、配置样板、密钥保护、AI Key、Docker、数据库表结构、pgvector 知识库、Redis、前端和后端是否正常

第一次双击时，如果系统提示不允许打开，可以右键文件，选择“打开”。启动窗口不要关闭，关闭后本地工作台也会停止。

## 命令行打开方式

如果电脑还没有 pnpm，先运行：

```bash
corepack enable
corepack prepare pnpm@9.15.4 --activate
```

安装依赖：

```bash
pnpm install
```

准备本地配置文件：

```bash
cp .env.example .env
```

启动数据库和队列：

```bash
docker compose up -d
```

如果这一步提示找不到 `docker`，说明电脑还没有安装或打开 Docker Desktop。你仍然可以先打开工作台体验写作流程；只是「保存为作品」会返回临时作品，等 Docker 可用后再保存才会真正写入本地数据库。

应用数据库表结构：

```bash
pnpm db:migrate
```

这一步会先准备数据库客户端，再把项目自带的表结构应用到本机 PostgreSQL；一般不会要求你回答命令行问题。

启动本地工作台：

```bash
pnpm dev
```

打开：

```txt
http://localhost:3000
```

后端接口会在：

```txt
http://localhost:3001/api
```

想快速检查本机环境时，可以运行：

```bash
pnpm local:doctor
```

它会检查配置文件、配置样板、密钥保护、AI Key、Docker、PostgreSQL、数据库表结构、pgvector 知识库、Redis、前端页面和后端接口，并用大白话告诉你下一步该处理什么。

想确认“自动写作主流程”有没有跑通时，可以在已启动项目后运行：

```bash
pnpm local:smoke
```

它会用一条测试灵感跑通真实自动写作链路、写作记忆/个人策略召回、保存作品、复盘预览，并自动删除临时作品和临时记忆。

也可以打开「设置中心」，在「写作主流程检查」里直接点「检查主流程」，页面会显示每一步是否正常。

## 配置 AI Key

打开 `.env`，然后按需填入：

```env
AI_PROVIDER=kimi
MOONSHOT_API_KEY=你的 Kimi Key
KIMI_TEXT_MODEL=kimi-k2.6
KIMI_BASE_URL=https://api.moonshot.ai/v1
NEXT_PUBLIC_API_URL=http://localhost:3001
```

也可以切到 DeepSeek：

```env
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的 DeepSeek Key
DEEPSEEK_TEXT_MODEL=deepseek-v4-flash
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

填好 Key 后，灵感写作、自动写作和局部改稿会优先尝试真实 AI。没有 Key 或调用失败时，系统会自动回到本地模拟内核，页面不会卡死。

你也可以在「设置中心」里直接选择 Kimi、DeepSeek 或 OpenAI，填写 API Key、写作模型和 Base URL，点击「保存设置」后会写入本机 `.env.local`，重启后仍然生效；点击「测试连接」可以查看当前 AI 状态。设置中心的本地运行体检已经折叠到下方，需要排错时再打开。

## 保存作品

在灵感写作或自动写作生成方案后，点击「保存为作品」。

- PostgreSQL 已启动：作品和完整写作方案会保存到本地数据库。
- PostgreSQL 未启动：作品会保存到本地文件 `storage/local-data/works.json`，重启后仍能在作品专栏看到。
- 无论数据库是否启动，保存生成方案时都会把测试读者报告里的首要建议写入写作记忆库，用来反哺下一篇。
- 保存成功后，可以直接点击「打开编辑器」继续修改正文。

作品专栏会优先读取后端数据库；如果数据库暂时不可用，会展示本地文件里的作品，并补充内置示例作品。顶部可以手动建档，也可以按草稿、已发布、连载中、已完结筛选，或按作品名、赛道、标签、平台搜索。
作品专栏里的单部作品可以编辑资料、维护阅读/收益/完读率/评论反馈，也可以删除；数据库未启动时，这些操作会落到本地文件。
作品专栏里的单部作品也可以点击「编辑」，直接进入对应作品的正文编辑器。
作品专栏里的单部作品还可以点击「详情」，进入作品详情页，集中查看数据表现、故事梗概、情绪曲线、冲突阶梯、信息差、人物关系、场景卡、提示词、分场正文、测试读者报告和修订建议；在「正文」页可以查看待处理标记和最近改写版本，也可以给某一场写反馈，然后点击「重写并保存本场」，系统会只改这一场并保存回整篇作品；在「记忆策略」页可以同时查看本作品相关的写作记忆和个人策略；在「复盘报告」页可以直接跳到复盘分析，并自动选中当前作品，也可以把复盘建议带去自动写作生成下一篇。

## 编辑器记录

正文编辑器会读取已保存作品，可以切换不同作品继续改稿。「保存正文」「添加标记」「删除标记」「@标记反馈」「生成局部改稿」「重新生成」「应用修改」已经接入后端。反馈框里输入 `@` 会弹出标记选择，选中后可以直接按这条标记生成改稿。应用修改前会让你确认本次是否更新写作记忆，并可选择人物状态、人物关系、已埋伏笔、已兑现伏笔、情绪弧线、用户偏好和后续注意事项；未勾选的内容不会进入记忆库。应用修改后，版本历史会保留原文、新文、修改原因和本次确认的记忆影响。

- PostgreSQL 已启动：标记、版本历史和改完的作品全文会保存到本地数据库；标记和版本历史也会同步留一份本地档案，方便导出作品工程。
- PostgreSQL 未启动：标记和版本历史会保存到本地文件 `storage/local-data/editor.json`，改完的作品全文会保存到 `storage/local-data/works.json`，重启后仍能读取。

## 写作记忆库和个人策略库

写作记忆库和个人策略库已经接入后端。

- 在编辑器里应用改稿时，会先让你确认是否把本次影响写入记忆；勾选的内容才会生成记忆。
- 在「复盘分析」里生成复盘后，会自动把复盘经验写入记忆，并把可执行的复盘结论写入个人策略库。
- 在「写作记忆库」页面可以读取后端记忆，按类型和关键词筛选，新增一条手动记忆，编辑、启用、停用或删除过时记忆。
- 同一页面也可以读取个人策略，按来源和关键词筛选，新增一条手动策略，编辑、启用、停用或删除过时策略；每条策略都可以直接点击「用它写下一篇」，把策略带入自动写作。
- 在「灵感写作」和「自动写作」生成前，系统会读取已启用的相关记忆和策略，让新方案参考你的个人偏好、复盘经验和下一篇行动规则。
- 在生成前，系统会把记忆和策略同步成一个本地知识库索引；Docker/PostgreSQL 可用并完成迁移时会写入 pgvector 向量索引，数据库不可用时写入 `storage/local-data/knowledge-index.json` 作为轻量兜底。
- 生成结果会显示「本次参考的写作记忆」，里面也会包含个人策略和知识库相似度，方便你确认它是否真的用上了经验。
- PostgreSQL 已启动时，写作记忆和个人策略会优先保存到本地数据库。
- PostgreSQL 未启动时，记忆会保存到本地文件 `storage/local-data/writing-memories.json`，个人策略会保存到 `storage/local-data/personal-strategies.json`，重启后仍能读取。

## 复盘分析

打开「复盘分析」，选择作品后可以录入发布后表现并生成复盘报告。

- PostgreSQL 已启动：复盘报告会保存到本地数据库。
- PostgreSQL 未启动：复盘报告会保存到本地文件 `storage/local-data/reviews.json`，重启后仍能读取。
- 生成复盘时，会优先读取作品里已导入的评论反馈和评论关键词，也可以手动补充阅读量、收益、完读率、排名变化、推荐量变化和评论反馈。
- 手动补充的阅读量、收益、完读率和评论反馈会同步回作品库，数据看板和作品详情会继续使用这次更新后的表现数据。
- 复盘报告会保存当次的表现数据快照和内容维度诊断，后续重启、备份和导出作品工程时不会只剩摘要。
- 复盘报告会拆成基础表现、内容表现判断、开头抓人程度、中段拖沓风险、高潮有效性、结尾情绪释放、人物记忆点、场景卡有效性、信息差发挥和冲突阶梯升级等维度。
- 有用的经验会同步写入写作记忆库，并生成会反哺下一篇写作的个人策略。

## 导出作品工程目录

打开「作品专栏」，点击单部作品的「导出工程」。

系统会在本地生成一套 Markdown 文件，默认位置是：

```txt
workspace/works/作品名
```

目前会导出真实正文、分场正文、人物卡、大纲、情绪曲线、冲突阶梯、信息差、场景卡、场景写作提示词、Agent 调度轨迹、测试读者报告、质量体检、原创边界、作品记忆、修订建议、最新发布复盘、标记改稿记录、写作记忆、个人策略和经验沉淀等文件；如果还没生成复盘，`review.md` 会写清下一步该去哪里补。若这部作品是由 Agent 生成后保存的，本地文件还会保留当时的原始写作方案，并在工程目录里额外导出 `source_plan.json`、`agent_trace.md`、`quality_report.md`、`originality_report.md`、`work_memory.md`、`revision_suggestions.md`、`marks.md`、`editor_history.json` 和 `memory_context.json`；每个场景的提示词会单独写入 `prompts/scene-xx.md`，每个场景的正文会单独写入 `drafts/scene-xx.md`，方便以后迁移、备份或做二次加工。

## 数据源和 CSV 导入

打开「数据源管理」，可以新增数据源，也可以直接读取公开网页、选择本地 CSV 文件、粘贴 CSV 内容、导入平台文字或截图。新增数据源时可以填写来源地址/文件说明、更新频率和是否启用；公开页面数据源保存后，可以直接在列表里点击「采集」。CSV 会先展示字段识别结果、字段映射和前几行样例，确认没问题后再导入。数据源列表和采集任务日志支持搜索、类型筛选和状态筛选，截图识别失败的任务会单独提示继续校正。

- 「番茄作者后台授权学习」支持启动本地浏览器采集：系统会打开一个临时浏览器窗口，用户自行登录、处理验证码和平台确认；停在作品数据、评论或活动页后，点击「读取当前页」即可把可见文字写入趋势、作品表现和下一篇创作依据。这个执行器不保存账号、密码或 Cookie；外部浏览器、截图和可见文字仍作为兜底入口。
- 授权可见页、CSV、截图校正和手动粘贴识别到作品表现或评论反馈后，会自动生成一条“平台表现记忆”和一条“下一篇个人策略”。同题材同规则已存在时会跳过，避免重复刷屏。
- 支持识别 `平台`、`题材`、`热度`、`增长率`、`机会分`、`饱和度`、`标签`、`原因`、`作品名`、`阅读量`、`收益`、`完读率`、`评论反馈`、`评论关键词` 这些列。
- 选择 CSV 文件后，页面会自动读取文件内容，识别常见字段别名，并展示前 5 行数据预览；也可以不选文件，直接把 CSV 内容粘贴进去。
- 如果平台导出的表头不标准，可以在「字段映射」里手动指定：例如把 `书名列` 映射成 `作品名`，把 `播放` 映射成 `阅读量`，把 `稿费` 映射成 `收益`。
- 选择本地 CSV 文件导入时，系统会把这份 CSV 原件保存到 `storage/uploads/csv`，采集日志会保留原件路径，方便以后回看这次导入来自哪份文件。
- 公开网页采集会读取外部公开页面文字，提取题材、标签、热度线索，生成趋势数据和采集日志；不会读取本机、内网、登录后或需要绕过验证的页面。
- 公开网页采集失败时，采集任务日志会显示失败原因和来源地址；只要系统保留了公开网址，就可以直接在日志里点「重试」重新采集。
- 如果 Redis 已启动，公开网页采集会优先进入本地 BullMQ 队列处理；如果 Redis 没启动，会自动改为直接采集，页面仍然可以正常使用。
- 如果 CSV 里带有 `作品名`、`阅读量`、`收益`、`完读率`、`收藏`、`评论反馈`、`评论关键词` 等列，系统会自动更新同名作品；找不到同名作品时，会创建一条来自平台导入的作品记录。
- 手动粘贴支持一行一个题材，也支持一行一部作品表现，例如：`作品名：她把旧姓还给雨夜 阅读量：12万 收益：438 完读率：71% 题材：女性成长 评论反馈：读者说女主反击克制但中段略慢 评论关键词：克制反击、节奏慢`。
- 导入后会生成一条采集任务日志，并同步追加到 `logs/crawler.log`。
- 数据源管理页会显示最近一次导入结果，包括识别行数、新增趋势、更新/新建作品数量，以及这次是通过本地 Redis 队列处理还是直接处理。
- 如果导入内容里有题材热度，会进入趋势接口，并在「风向标」页面展示。
- CSV、手动粘贴、公开网页和截图校正导入的数据会保留来源标签；风向标、作品专栏和作品详情会显示这条数据来自哪次导入，评论反馈和评论关键词在 PostgreSQL 可用时也会写入本地数据库。
- 截图导入会把原图保存到 `storage/uploads/screenshots`。如果同时粘贴 OCR 文字或手动校正文字，系统会立刻识别题材趋势和作品表现；如果只上传截图不填文字，系统会在已配置 Kimi/OpenAI Key 时自动读图提取文字，再写入趋势数据、作品库和采集日志；DeepSeek 当前只用于文本写作和分析；没有可读图 Key 或识别不到可用文字时，会生成一条“等待校正”的采集日志，后续可以在等待截图卡片里直接粘贴校正文字，不需要重新上传原图。
- PostgreSQL 已启动：数据源和日志会保存到本地数据库，来源 URL、CSV 原件路径和截图路径会作为独立字段保存，方便后续追溯和重试。
- PostgreSQL 未启动：数据源和采集日志会保存到 `storage/local-data/datasources.json`，公开网页、CSV、平台文字或截图自动识别/校正文字生成的趋势会保存到 `storage/local-data/trends.json`，CSV 原件会保存在 `storage/uploads/csv`，截图文件会保存在 `storage/uploads/screenshots`，重启后仍能读取。

## 数据看板和趋势写作联动

「数据看板」会读取后端作品和趋势数据，汇总阅读量、订阅量、收益、平均完读率、当前强势赛道、评论关键词和标签表现。它还会把平台趋势拆成榜单样本、活动/征文窗口和题材机会表现，同时拆解高完读/高收益作品，提示疑似章节/场景流失点，并用赛道收益、完读率风险、创建/导入日期表现帮助判断下一篇该写什么。CSV、平台文字或截图自动识别/校正文字导入作品表现后，这些数字和评论关键词会跟着刷新。

在「风向标」或首页推荐里点击对应题材，会自动打开「自动写作」，并带入当前题材、平台和推荐理由。

「自动写作」右侧会读取后端 TOP 趋势，也可以点击「套用这个建议」把最新强势赛道写入创作参数。

## 本地备份

打开「设置中心」，点击「导出全部数据」。

设置中心会先展示当前本地存储和采集设置，方便确认数据保存在自己的电脑上；备份与恢复按钮在同一页。

系统会把作品、趋势、写作记忆、个人策略、知识库索引、数据源、采集日志、复盘报告、正文标记和版本历史导出为一个 JSON 文件，默认位置是：

```txt
storage/backups
```

备份文件不会包含 `MOONSHOT_API_KEY`、`DEEPSEEK_API_KEY`、`OPENAI_API_KEY` 等密钥原文。

「恢复最新备份」会读取 `storage/backups` 里的最新备份文件。有 PostgreSQL 时会写回本地数据库；数据库未启动时，会写回 `storage/local-data` 里的本地数据文件。

「清理导入/验证数据」会先自动导出一份备份，再清理公开网页/CSV/截图/手动粘贴导入记录、导入趋势、平台导入生成的作品、标题或规则里带“验证/测试”的临时数据、临时记忆、临时策略，以及截图缓存。它不会清理正常保存的创作作品。

「清理运行缓存」只会清理截图缓存、临时缓存和运行缓存；「清空日志」只会清空本地日志目录。它们都不会删除作品、备份、写作记忆和个人策略。

「重置为初始数据」会先自动导出一份备份，再把作品、趋势、写作记忆、个人策略、知识库索引、复盘、正文标记、版本历史和数据源恢复成刚安装时的示例状态。这个按钮适合想重新开始整理工作台时使用。

## 重要边界

本产品只做合规创作辅助：公开数据分析、用户授权数据导入、公开网页/CSV/截图导入、原创写作、改稿、复盘和写作记忆沉淀。

不做绕过登录、绕过验证码、绕过平台风控、规避 AI 检测、复制热门作品全文、批量低质水文或自动发布。
