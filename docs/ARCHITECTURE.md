# 本地运行架构

```txt
用户浏览器
  ↓
apps/web：Next.js 工作台
  ↓
apps/api：NestJS API 服务
  ↓
PostgreSQL / Redis / 本地文件目录
```

## 目录

```txt
apps/
  web/         前端工作台
  api/         后端 API
packages/
  shared/      共享类型、mock 数据、写作内核
storage/       上传、导出、截图
workspace/     作品工程目录
logs/          本地日志
docs/          产品与开发文档
```

## 第一版实现边界

- UI 和交互完整跑通
- API 服务可启动
- Prisma schema 和 PrismaService 已接入
- 作品保存、手动建档、资料编辑和删除接口已接入 PostgreSQL，数据库不可用时会写入 `storage/local-data/works.json`，并保留当次生成的原始写作方案；本地文件会记录已删除作品，避免回退模式下反复显示
- 标记和版本历史接口已接入 PostgreSQL，数据库可用时也会同步本地档案，数据库不可用时会写入 `storage/local-data/editor.json`
- 正文编辑器会读取真实作品全文，支持 @ 标记反馈、局部改稿重新生成、应用修改和版本对照；应用局部改稿后会把改完的全文回写到作品，并可按用户勾选项选择是否更新写作记忆
- 已保存作品的详情页也能按单个场景重写正文，并把新的分场稿、整篇正文和读者建议保存回作品
- 写作记忆接口已接入 PostgreSQL，数据库不可用时会写入 `storage/local-data/writing-memories.json`；个人策略库会写入 `storage/local-data/personal-strategies.json`
- 应用改稿时可确认是否沉淀人物状态、人物关系、伏笔、情绪弧线、用户偏好和后续注意事项，复盘会额外沉淀个人策略
- 灵感写作和自动写作会在生成前读取已启用写作记忆和个人策略，生成结果会展示本次参考的经验
- 灵感写作可以保存灵感，自动写作可以保存参数模板，记录写入 `storage/local-data/writing-assets.json` 并进入本地备份
- 生成结果支持步步确认，可先确认选题卡，再查看结构、人物卡、场景卡、提示词、分场正文草稿、测试读者报告和修订建议；选中另一张选题卡后可重新生成后续方案
- 写作内核新增分阶段工作流合同：场景卡、场景提示词、分场正文必须按 sceneId 和顺序一一对应；整篇 `draft` 必须由 `sceneDrafts` 合并而来；Agent 轨迹固定为主控、风向、选题、结构、场景卡、提示词、正文、测试读者、编辑改稿、复盘沉淀 10 步
- 真实 OpenAI 返回的方案会先经过 `enforceStoryWorkflow` 和 `validateStoryWorkflow`，避免模型另写一版与分场正文不一致的全文
- 写作内核会为每张场景卡生成一条独立写作提示词和一段分场正文，结果页可查看分场小评分和修改重点，并支持保存前只重写某一场；保存后在作品详情里仍可继续只重写某一场；导出工程会写入 `scene_prompts.md`、`prompts/scene-xx.md`、`scene_drafts.md` 和 `drafts/scene-xx.md`
- 复盘报告接口已接入 PostgreSQL，数据库不可用时会写入 `storage/local-data/reviews.json`；报告会保存表现数据快照和内容维度诊断，避免重启或导出时丢失复盘细节
- 复盘分析支持录入阅读、收益、完读率、排名变化、推荐量变化和评论反馈；录入的核心表现会同步回作品库，报告会生成基础表现和内容维度诊断，包括开头抓人程度、中段拖沓风险、高潮有效性、结尾情绪释放、人物记忆点、场景卡有效性、信息差发挥和冲突阶梯升级
- 生成复盘后会自动沉淀记忆和个人策略，并会优先读取作品里已导入的评论反馈与评论关键词
- 作品工程目录导出会写入 `workspace/works`，优先使用真实正文、分场正文和已保存的原始写作方案，并单独导出修订建议、最近一次发布复盘、标记改稿记录、真实写作记忆、个人策略、`editor_history.json` 和 `memory_context.json`
- 数据源和采集日志已接入 PostgreSQL，数据库不可用时会写入 `storage/local-data/datasources.json`
- 公开网页采集、CSV 文件选择/粘贴导入、平台文字粘贴导入和截图自动识别/校正文字导入已接入，新增数据源支持来源地址/文件说明、更新频率和启用状态；来源 URL、CSV 原件路径和截图路径会作为独立字段写入数据源和采集日志，旧备注里的来源仍可兼容读取；公开页面数据源可从列表直接运行采集，CSV 会在前端先做字段识别、字段映射和样例预览；选择本地 CSV 文件导入时会把原件保存到 `storage/uploads/csv`，公开网页失败日志可保留来源地址并一键重试，生成的趋势数据在数据库不可用时会写入 `storage/local-data/trends.json`
- 采集任务会同步追加到 `logs/crawler.log`
- CSV、平台文字和截图自动识别/校正文字可识别作品名、阅读量、收益、完读率、收藏、评论反馈、评论关键词等字段，并回写作品库
- CSV 原件和截图原图都已接入本地文件保存：CSV 写入 `storage/uploads/csv`，截图写入 `storage/uploads/screenshots`；截图带 OCR 或手动校正文字时会解析题材趋势和作品表现，只上传图片且已配置 OpenAI Key 时会自动读图提取文字，没有 Key 或识别不到可用文字时会生成等待校正的采集日志，后续可直接给这条等待任务补充校正文字
- 风向标页面已接入后端趋势接口，能展示 CSV 导入的新题材，并把趋势拆成热门赛道、活动/征文方向、同质化风险提醒、题材机会卡和榜单作品拆解
- 数据看板已接入后端作品和趋势数据，可汇总阅读、订阅、收益、完读率、赛道收益对比、完读率风险、评论关键词和创建/导入日期表现，并补充平台榜单拆解、活动/征文窗口、题材机会表现、高完读/高收益作品拆解和疑似章节/场景流失点
- 风向标机会卡可带题材、平台和推荐理由进入自动写作
- 首页创作驾驶舱已接入后端作品和趋势数据，今日灵感可切换并带入灵感写作页，本周创作进度会按草稿/连载作品的真实更新时间和字数计算，并跳转到最近可继续写的作品
- 自动写作右侧风向标建议已接入后端 TOP 趋势
- 生成结果保存后和作品专栏都可以带作品编号进入正文编辑器
- 作品专栏支持手动建档、编辑资料、删除作品、按作品状态筛选，并按作品名、赛道、标签或平台搜索
- 正文编辑器的反馈框支持输入 `@` 选择标记，并可直接按所选标记生成局部改稿
- 写作记忆库支持按来源类型筛选、关键词搜索、手动新增、编辑、启用、停用和删除
- 个人策略库支持按策略来源筛选、关键词搜索、手动新增、编辑、启用、停用和删除
- 作品专栏可以进入单部作品详情页，详情页读取真实作品，展示数据表现、故事梗概、情绪曲线、冲突阶梯、信息差、人物关系图谱、场景卡、场景提示词、分场正文、测试读者报告、修订建议和正文草稿，并支持单场重写后保存
- 设置中心已接入 AI 状态、AI Key/写作模型/Embedding 模型保存、本地运行体检、本地存储目录、采集设置、采集边界、本地 JSON 备份导出、最新备份恢复、运行缓存清理和日志清空；AI 设置会写入本机 `.env.local`，体检会检查 PostgreSQL、Redis、pgvector 知识库、文件存储、作品工程目录和日志目录；备份包含作品、复盘、写作记忆、个人策略、数据源和趋势，输出到 `storage/backups`，无数据库恢复会写回 `storage/local-data`
- Docker Compose 已提供 PostgreSQL/pgvector 和 Redis
- 写作内核支持 OpenAI/本地模拟双模式
- 数据采集遵守公开页面和用户授权导入边界
